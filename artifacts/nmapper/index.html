🧮 Méthodologie de scoring du risque

Chaque hôte reçoit un score de risque pondéré de 0 à 100, calculé de façon entièrement déterministe et traçable — aucune IA, aucune estimation aléatoire. Le score additionne des points selon des facteurs observés directement dans les résultats Nmap (ports ouverts, scripts NSE, versions de services, système d'exploitation), puis il est plafonné à 100. Le détail exact des points attribués à un hôte est visible dans son popup de détails, section « 🧮 Détail du calcul ».

FacteurPointsDétail
Ports critiques (OT/ICS, non chiffré) jusqu'à +40 Modbus 502, S7 102, OPC-UA 4840, BACnet 47808, DNP3 20000, EtherNet/IP 44818, Telnet 23… (+22 chacun, plafonné)
Ports à risque élevé jusqu'à +20 SNMP 161/162, Elasticsearch 9200, VNC 5800… (+10 chacun)
Ports à risque moyen jusqu'à +15 SSH 22, SMB 445, RDP 3389, HTTP 80/8080, FTP 21… (+4 chacun)
Services non chiffrés jusqu'à +15 FTP, Telnet, HTTP, SNMP (21/23/80/8000/8080/161) — +5 chacun
Findings NSE (scripts Nmap) +10 à +40 Critique +40 (ex. smb-vuln-ms17-010 VULNERABLE + CVE), élevé +25 (ftp-anon, heartbleed…), moyen +10 (TLS faible, RC4/MD5…)
Versions de service obsolètes +8 à +30 Critique +30 (vsftpd 2.3.4 backdoor), élevé +18 (Exim < 4.92 RCE, OpenSSL 1.0.x…), moyen +8 (OpenSSH < 7.0, Apache < 2.4…)
OS en fin de vie (EOL) +10 à +30 Critique +30 (Windows XP/2000/Vista), élevé +20 (Windows 7, Server 2003/2008, noyau Linux 2.x), moyen +10 (Windows 8, Server 2012…)
Surface d'exposition +5 à +10 ≥ 6 ports ouverts : +5 · ≥ 10 ports ouverts : +10
Escalade sémantique : un finding confirmé impose un plancher de gravité, quel que soit le total arithmétique. Exemple : un script NSE renvoyant VULNERABLE avec une CVE (type EternalBlue) classe l'hôte en Critique même si la somme des points resterait sous 70. À l'inverse, le score reste plafonné à 100. Tous les facteurs proviennent de faits Nmap vérifiables — le scoring est reproductible et auditable.

📋 Installation manuelle

Commandes à copier-coller directement sur la machine cible.

🔑 Clé API : chargement…
🟢 Prérequis : Python 3 uniquement — aucun pip install nécessaire.
① Copiez agent.py sur la machine cible
scp installation/agent.py root@TARGET_IP:/opt/nmapper-agent/agent.py
② Lancez l'agent (test immédiat)
python3 /opt/nmapper-agent/agent.py \ --server-ip NMAPPER_IP \ --port 25774 \ --key API_KEY_PLACEHOLDER \ --interval 30
③ (Optionnel) Service systemd — démarrage automatique
cat > /etc/systemd/system/nmapper-agent.service << 'EOF' [Unit] Description=NMAPPER Agent de collecte After=network-online.target Wants=network-online.target [Service] ExecStart=python3 /opt/nmapper-agent/agent.py --server-ip NMAPPER_IP --port 25774 --key API_KEY_PLACEHOLDER --interval 30 Restart=always RestartSec=15 [Install] WantedBy=multi-user.target EOF systemctl enable --now nmapper-agent
🔵 Prérequis : Python 3.8+ et pip.
① Copiez les fichiers backend sur le serveur
scp -r python-api/* root@TARGET_IP:/opt/nmapper/
② Installez les dépendances Python
pip3 install fastapi 'uvicorn[standard]' slowapi bcrypt \ pyotp 'qrcode[pil]' pillow python-multipart pydantic starlette
③ Créez le dossier de stockage et lancez le backend
mkdir -p /opt/nmapper/scans cd /opt/nmapper && python3 -m uvicorn main:asgi_app \ --host 0.0.0.0 --port 25774
④ (Optionnel) Service systemd
cat > /etc/systemd/system/nmapper-server.service << 'EOF' [Unit] Description=NMAPPER Backend API After=network.target [Service] WorkingDirectory=/opt/nmapper ExecStart=python3 -m uvicorn main:asgi_app --host 0.0.0.0 --port 25774 Restart=always RestartSec=5 [Install] WantedBy=multi-user.target EOF systemctl enable --now nmapper-server
💡 La clé API est générée automatiquement au premier démarrage dans /opt/nmapper/scans/.apikey
